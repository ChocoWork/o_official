# OWASP ASVS v4.0.3 チェックリスト（Web アプリケーション Level 2 抜粋）

ASVS は Verification Level 1 (Opportunistic) / 2 (Standard) / 3 (Advanced) の3段階。商用 Web アプリは **Level 2** が最低基準。

## V2: Authentication

### V2.1 Password Security
- [ ] V2.1.1: パスワード最低 12 文字
- [ ] V2.1.2: パスワード最大 128 文字以上を許可
- [ ] V2.1.3: 全 Unicode 文字をサポート
- [ ] V2.1.7: 既知の漏洩パスワードチェック（Have I Been Pwned API 等）
- [ ] V2.1.9: パスワード強度メーター提供
- [ ] V2.1.11: パスワードペースト機能を妨げない

### V2.2 General Authenticator
- [ ] V2.2.1: 認証はサーバ側で実施（クライアント側のみで完結させない）
- [ ] V2.2.3: 認証情報は ASVS 準拠の暗号でハッシュ化
- [ ] V2.2.5: ブルートフォース対策（レート制限/アカウントロック）

### V2.4 Credential Storage
- [ ] V2.4.1: Argon2id（メモリ ≥ 19MiB、反復 2、並列度 1）または
- [ ] V2.4.2: scrypt（N=2^17, r=8, p=1）または
- [ ] V2.4.3: bcrypt（cost ≥ 10、推奨 12）または
- [ ] V2.4.4: PBKDF2 (iterations ≥ 600,000 for SHA-256)
- [ ] V2.4.5: パスワードに salt が付与されている

## V3: Session Management

- [ ] V3.2.1: セッショントークンはサーバー側で生成
- [ ] V3.2.2: 64bit 以上のエントロピー
- [ ] V3.2.3: セッションを Cookie で扱う場合、Secure; HttpOnly; SameSite を設定
- [ ] V3.3.1: ログアウトでセッション完全終了
- [ ] V3.3.2: 非アクティブタイムアウト（推奨 30 分）
- [ ] V3.3.3: 絶対タイムアウト（推奨 12 時間）
- [ ] V3.4.1: Cookie に `Secure` 属性
- [ ] V3.4.2: Cookie に `HttpOnly` 属性
- [ ] V3.4.3: Cookie に `SameSite` 属性（Lax または Strict）
- [ ] V3.4.4: Cookie の `Path` 属性で範囲限定
- [ ] V3.4.5: 機密 Cookie は `__Host-` プレフィックス
- [ ] V3.5.2: ステートレストークン（JWT）の場合、署名検証必須
- [ ] V3.5.3: JWT の `alg: none` 拒否

## V4: Access Control

- [ ] V4.1.1: 信頼境界をサーバー側で強制
- [ ] V4.1.2: ユーザー属性は信頼できるソースから取得（クライアントから受け取らない）
- [ ] V4.1.3: 「最小権限の原則」
- [ ] V4.1.5: アクセス制御失敗時のフェイルセーフ（Deny by default）
- [ ] V4.2.1: IDOR 対策（リソース所有者検証）
- [ ] V4.2.2: 状態変更操作に CSRF トークンまたは SameSite Cookie
- [ ] V4.3.1: 管理機能に多要素認証
- [ ] V4.3.2: ディレクトリブラウジング無効

## V5: Validation, Sanitization, Encoding

- [ ] V5.1.1: HTTP パラメータ汚染対策
- [ ] V5.1.2: マスアサインメント対策（許可リスト方式）
- [ ] V5.1.3: すべての入力に正のバリデーション（Zod 等）
- [ ] V5.1.4: 構造化データの型・範囲・長さ検証
- [ ] V5.1.5: リダイレクト URL は許可リスト
- [ ] V5.2.1: 信頼できない HTML は DOMPurify 等でサニタイズ
- [ ] V5.2.2: Markdown の生 HTML を安全にレンダリング
- [ ] V5.2.3: メール本文に SMTP/IMAP インジェクション対策
- [ ] V5.2.4: `eval`、動的コード実行を使わない
- [ ] V5.2.5: テンプレートインジェクション対策
- [ ] V5.2.7: SVG 入力のスクリプト除去
- [ ] V5.3.1: コンテキストごとの出力エンコーディング
- [ ] V5.3.3: コンテキスト対応 XSS 対策
- [ ] V5.3.4: SQL インジェクション対策（パラメータ化）
- [ ] V5.3.5: NoSQL インジェクション対策
- [ ] V5.3.6: LDAP インジェクション対策
- [ ] V5.3.7: OS コマンドインジェクション対策
- [ ] V5.3.8: XML インジェクション対策（XXE 含む）
- [ ] V5.3.10: XPath インジェクション対策

## V7: Error Handling and Logging

- [ ] V7.1.1: パスワード/トークン/PII をログに含めない
- [ ] V7.1.2: 機密データのログマスキング
- [ ] V7.1.3: セキュリティイベントの記録
- [ ] V7.1.4: ログに適切なメタデータ（時刻、ユーザーID、IP）
- [ ] V7.2.1: セキュリティ関連ログを集中管理
- [ ] V7.4.1: 汎用エラーメッセージを返却（スタックトレース禁止）

## V8: Data Protection

- [ ] V8.1.1: ブラウザに機密データをキャッシュさせない（`Cache-Control: no-store`）
- [ ] V8.2.1: 認証ページに `Cache-Control: no-store`
- [ ] V8.3.1: 機密データを GET の URL パラメータに含めない
- [ ] V8.3.4: 機密データの保持期間を定義し最小化

## V9: Communications

- [ ] V9.1.1: TLS 1.2 以上
- [ ] V9.1.2: 弱い暗号スイート無効化
- [ ] V9.1.3: HSTS 設定（max-age ≥ 1 年、includeSubDomains 推奨）
- [ ] V9.2.1: サーバ証明書の検証

## V10: Malicious Code

- [ ] V10.1.1: アンチウイルススキャン（ファイルアップロード時）
- [ ] V10.2.1: アプリケーションが想定外コードを実行できない
- [ ] V10.3.2: 完全性検証（SRI、署名）
- [ ] V10.3.3: タイムボム/ロジックボムが含まれていない
