## スコープ判断（重要・ここで「あえて全ページ均一レビューに従わない」理由）

`/ui` は全UIコンポーネント（Accordion〜Tooltip まで40+）を一覧表示する**開発者向けコンポーネントギャラリー**（[ui/page.tsx:8-46](../../src/app/ui/page.tsx#L8-L46) で全primitiveをimportしデモ表示）。これは顧客導線上のページではなく、内部の確認用ツールであり、性質は admin（レビュー対象外）に近い。

> ご依頼は「Admin以外の全ページ」だが、本ページに**顧客体験としての詳細UI/UXレビューを行うのは費用対効果が低い**と判断した。理由:
> 1. 顧客のタスク（閲覧・購入・問い合わせ）に一切関与しない内部ツールで、ペルソナの誰も到達しない想定。
> 2. ブランド世界観の表現対象ではなく、各コンポーネントの仕様確認が目的。
> 3. むしろ**本番に存在・公開ナビに露出していること自体が問題**であり、レビューより「公開からの除外」が正しい打ち手。

この判断にご納得いただけない場合は、通常ページと同じ粒度で再レビューする。

## 修正反映（2026-06-23）

- **UI-1**: Header の `menuItems` から `{ href: '/ui', label: 'UI' }` を削除（公開ヘッダー/ドロワーから除外）。home C-1 と同件で解消。
- **UI-2**: `ui/page.tsx` 冒頭で `process.env.NODE_ENV === 'production'` のとき `notFound()` を呼び、本番ビルドでは 404（到達不可）に。開発環境のみ閲覧可能。
- **UI-3**: ギャラリー内の状態色は全バリアント確認用として妥当なため**変更不要**と判断（実ページ採用時はモノクローム逸脱判断に従う、という運用方針を確定）。

## 総評（/ui）

UI/UXの良否以前に、**内部ツールが本番・公開ナビに出ている運用面**が論点。除外（UI-1/UI-2）を推奨。コンポーネント群自体の品質は高く、これが各ページの良い土台になっている。
