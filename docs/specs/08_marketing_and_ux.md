## 概要
- マーケティングとUXに関する機能（パーソナライズ、プロモーション、メール連携、ポップアップ等）をまとめる。

## 範囲 (In / Out)
- 含むもの: プロモーション管理、クーポン、ウィッシュリスト、パーソナライズ（ルールベース）、入荷通知、メール連携
- 含まないもの: マーケティングオートメーションの高度なワークフロー（別途連携）

## 機能要件
1. プロモーション管理: 管理画面でクーポン・セール設定を行えること（開始/終了日時、適用条件）
2. ウィッシュリスト: ユーザが商品を保存できること（匿名は Cookie、会員は DB 保存）
3. パーソナライズ: ルールベースのおすすめ（購入履歴や閲覧履歴に基づく）を MVP で導入
4. 入荷通知: 在庫切れ商品のメール通知登録
5. メール連携: SendGrid 等で注文/通知メールを送信

## 非機能要件
- メール配信の到達率・遅延をモニタリング
- プロモーション適用は原則サーバ側検証で整合性を担保

## API / インターフェース
- POST `/api/promotions` - プロモーション作成（管理 API）
- POST `/api/wishlist` - ウィッシュリスト追加
- POST `/api/items/:id/notify` - 入荷通知登録

## テストケース（受け入れ基準）
- プロモーションが正しく割引を反映すること（境界条件含む）
- ウィッシュリストの同期（匿名→会員）でデータ保持が行われること

## 実装ノート
- パーソナライズはまずサーバサイドでのルール評価を実装し、将来の ML 強化に備える
- メールテンプレートはブランドガイドに基づいて作成

## チケット分割例
- TASK-1: プロモーション管理 API 実装
- TASK-2: ウィッシュリスト UI + API 実装
- TASK-3: 入荷通知フロー実装
